# Case 005 — 20260713_182249.jpg (pin-loaded stack)

- **Source:** device gallery photo (Galaxy A52), pulled over MTP. STILL photo, not a clip — full
  sharpness, well-lit, painted white-on-black numbers. The *easy* case, on paper.
- **Equipment:** selectorized (pin-loaded) weight stack. Printed ladder: adder `5 KG`, then main stack
  `13, 21, 29, 37, 45, 53, 61, 69, 77, 85, 93, 101, 109, 117, 125, 133, 141, 149, 157, 165 KG`
  (20 plates, **+8 kg per plate**, base 13).
- **Task:** WEIGHT only — read the selected weight.

## Ground truth (lifter-confirmed)
- **37 kg.** Pin in the 4th plate from the top; **3 empty holes above it (13, 21, 29 open)**.

## What I predicted — 🔴 WEIGHT MISS ×4 then corrected (worst single-image failure)
`45 → 53 → 61 → 69 kg` (four passes, every one **too high**), plus a brief wrong `29 kg`. Only got
**37 kg** after the lifter spelled out the method: *"use the original image, just zoom it and count the
number of empty holes from the top."*

## Diagnosis — why a clean painted OCR target beat me four times
1. **Eyeballed the knob column, not the holes.** I kept anchoring on where the **domed selector knob**
   appeared to line up against the label strip. The knob is a **wide, offset, perspective-sheared body**
   — it visually overhangs plates 3–5 heavier than the one its stem actually enters → every guess landed
   too high.
2. **Label strip ≠ hole row alignment.** The numbers and the hole row sit at **different heights**;
   photographed at an angle, the two shear apart, so "the label above the knob" is not the pinned plate.
3. **Mistook the sticker border for a selector.** Each number sits in a rounded-rectangle **sticker
   outline**; I read the box around `29` as a "selected" highlight. It is on **every** label.
4. **Never counted the actual empty holes** until told to. The open (empty) holes above the pin are the
   unambiguous signal: 3 open → pin in plate 4 → 37 kg.
5. **Tooling self-own.** `convert -auto-orient` rotated the Samsung JPEG to **portrait**, so my crop
   offsets (tuned to the landscape view I was looking at) pointed at the label band instead of the hole
   row for several passes — wasted crops that looked "unreadable" for the wrong reason.

## The method that works (banked to weight-reading.md + skill)
1. Crop **labels + holes into ONE aligned frame**, without `-auto-orient`, and upscale.
2. Trace each plate's vertical strip from its number down to its hole so you know which plate hole #1 is.
3. **Count EMPTY (open, dark) holes from the TOP** (lightest) to the pin. `plate index = empty + 1`.
4. **weight = first-plate value + increment × (empty holes)**. This stack: `13 + 8 × 3 = 37 kg`. ✅
5. Never eyeball the knob-vs-label column; never treat the sticker border as a selection.
